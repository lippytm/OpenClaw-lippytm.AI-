# Agent Training Patterns

This document defines reusable training patterns for agent-oriented AI systems inside the wider `lippytm` ecosystem.

## Purpose
Create repeatable patterns for teaching, testing, and improving agent workflows through repository-based experimentation.

## Core Pattern
1. Define the agent role clearly
2. Define the environment or task context
3. Define the expected inputs and outputs
4. Run a small controlled experiment
5. Record observations and failures
6. Refine prompts, workflows, or structure
7. Convert the improved version into reusable documentation

## Pattern Types

### 1. Role Pattern
Use for:
- assistant roles
- planner roles
- researcher roles
- builder roles
- reviewer roles

### 2. Workflow Pattern
Use for:
- intake flows
- routing flows
- evaluation flows
- execution loops
- reflection loops

### 3. Safety Pattern
Use for:
- scope boundaries
- human review points
- risky action checkpoints
- documentation of failure cases

### 4. Teaching Pattern
Use for:
- turning agent experiments into lessons
- turning lessons into templates
- turning templates into reusable kits

## Best Practices
- start with narrow experiments
- keep inputs and outputs explicit
- separate production patterns from sandbox patterns
- document what failed as well as what worked
- link experiments back to teaching materials and builder tracks
